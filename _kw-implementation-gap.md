## B. h3 追加提案（ロングテール展開・優先度高）

設計書v1の段階で言及はあったが、実装で h3 まで落ちていない。Google検索でCV直結の高確率KW。

### B-1. ブランド × モデル別買取実績 h3（10個追加）
現状: hero rate表示で「バーキン25 / サブマリーナ / マトラッセ ラージ / デカマトラッセ30」の4モデルのみ
追加候補:
- **ロレックス サブマリーナの買取実績**（業界標準、Suggest確定の高ボリューム）
- **ロレックス デイトジャストの買取実績**（10社中9社で扱い）
- **ロレックス デイトナの買取実績**（高単価・需要大）
- **エルメス バーキン30/35の買取実績**（30 / 35サイズ別で2h3）
- **エルメス ケリー28の買取実績**（バーキン同等の人気）
- **シャネル マトラッセ25/デカマトラッセ30の買取実績**（既存hero rateの h3 化）
- **ヴィトン モノグラム長財布の買取実績**（**Suggest確定**「ヴィトン モノグラム 長財布 買取」）
- **ヴィトン ダミエの買取実績**（モノグラムと並ぶ二大定番）
- **オメガ スピードマスターの買取実績**（時計第2軸）
- **カルティエ サントス/タンクの買取実績**（時計+ジュエリー）

### B-2. エリア専用 h3（残り12区を簡略補強）
現状: 中村区/中区/千種区/名東区の **h3 が4つ**、残り12区は段落内で1行列挙
追加候補（**強化**は4区分のうち、自社近隣を優先）:
- **西区のブランド買取**（自社中村区から1区圏内、Suggest検索量あり）
- **中川区のブランド買取**（同上）
- **北区のブランド買取**（同上）
- **緑区のブランド買取**（自社既存LPで実績あるエリア）
- **天白区のブランド買取**（高単価層）
- 残り 7区は段落維持で良い

### B-3. シナリオ h3（顧客文脈ロングテール、4個追加）
- **相続したブランド品の買取**（既に「ご相談の典型例」h2 はあるが、相続専用 h3 なし）
- **形見整理のブランド品買取**（同上）
- **生前整理に伴うブランド品買取**（自社遺品整理LP連携）
- **引っ越し前のブランド整理**（カテゴリ実態）

### B-4. チャネル深掘り h3（4個追加）
- **LINE査定の使い方（写真の撮り方）**（業界標準、検索量大）
- **出張買取の流れ（訪問前/訪問時/訪問後）**（buysell の3ステップ）
- **宅配買取の流れ（キット手配〜入金まで）**（komehyo型）
- **テレビ電話・Zoom査定**（brandrevalue型、追加チャネル）

### B-5. 比較表 h2 / h3（業界0%、AEO最強差別化）
業界10社全社で実装ゼロ。これを実装すると **AI検索（Perplexity/ChatGPT）の引用率が跳ね上がる**。
- **h2「徳丸商会 vs ナショナル買取業者」** — 営業時間/許可種別/総合店性で比較
- **h2「買取 vs 質屋・フリマアプリ」** — 手間/速さ/価格で比較
- **h3「徳丸商会 vs 名古屋市内の他店」** — 12時間営業の優位を示す比較

合計 **新規 h3 約 25個**追加（実装規模: 各h3 + 50-100字本文 = 1,500-2,500字）

## C. FAQ追加提案（10問→15-18問に拡張）

現状の `data/faq.ts` は10問あるが、v2マップで Suggest 確定した検索意図がカバー出来ていない。

### 追加すべきFAQ 5-8問

1. **「ブランド品の偽物かもしれない場合でも査定してもらえますか？」**
   - 共起語: 真贋 / 真贋判定 / 鑑定士 / ダブルチェック
   - 業界共通の不安、AEO引用率高い

2. **「保証書（ギャランティカード）がない場合、買取価格はどれくらい下がりますか？」**
   - 既存「保証書なしOK」より一段深い。**Suggest確定**「ブランド 保証書なし 買取」
   - 「保証書ありで○%、なしで×%」のような数字的回答が AI引用率高い

3. **「メルカリやヤフオクで自分で売るのと、徳丸商会の買取はどちらが得ですか？」**
   - **fuku-chan**「フリマアプリより手軽でした」訴求の自社版
   - 比較表代わりとして AEO に強い

4. **「電池が止まっているブランド時計でも買取できますか？」**
   - 時計特有の不安、業界標準FAQ

5. **「テレビ電話やZoomでの査定はできますか？」**
   - **brandrevalue** が訴求、業界差別化

6. **「夜21時まで営業ですが、土日や祝日も同じですか？」**
   - 自社差別化軸の補強

7. **「即日で現金または振込してもらえますか？」**
   - **brandouro**「即日入金」、検索意図 **Suggest確定**「ブランド買取 即日」

8. **「初めての買取で不安です。どんな流れですか？」**
   - 一般的な初回不安、AEO 引用しやすい

合計 **15-18問**に拡張。各回答は最低100字、AEO狙いで150-200字推奨。

## D. 構造化データ（JSON-LD）追加提案

現状9種実装（業界先行者）。さらに 2-3種で完璧化。

### D-1. ItemList（買取実績一覧）
ロングテールで効く。`src/data/rates.ts` から自動生成可能。
```json
{
  "@type": "ItemList",
  "name": "ブランド買取実績",
  "itemListElement": [
    {"@type": "Product", "name": "エルメス バーキン25", "offers": {"@type": "Offer", "price": "1400000"}},
    ...
  ]
}
```

### D-2. Question / Answer（FAQ以外の段落QA）
比較表セクションをQA形式で実装すると、検索結果の **People Also Ask** に表示される可能性。

### D-3. Review（個別レビュー）
現状 `voices.ts` から FAQ的に出してるが、独立 `Review` 型 + `aggregateRating` を出すと **星評価** がGoogle検索結果に表示。
- ただし `aggregateRating` には**実数値・実集計が必須**（虚偽はNG。memory `feedback_review_protocol.md` 違反）
- 実集計運用が無いなら **保留**（やるならGoogle Business Profile からの集計を引く）

## E. AEO（生成AI検索）強化提案

speakable は実装済み。さらに以下を追加すると Perplexity/ChatGPT 引用率が上がる。

### E-1. 用語定義段落（300-500字）
- **「ブランド買取とは」段落** — リードで書く
- **「出張買取とは」段落** — チャネルセクション内
- **「LINE査定とは」段落** — チャネルセクション内
- **「徳丸商会とは」段落** — フッター上

→ AI が「ブランド買取の定義は？」と聞かれた時に、自社の段落をそのまま引用しやすくなる。

### E-2. 数字付き具体段落
- 「営業時間は **9:00-21:00、年中無休**（年末年始除く）」
- 「対応エリアは **名古屋市16区＋尾張・知多・西三河**、最短当日訪問」
- 「許可番号は **愛知県公安委員会 第541042309800号**」

→ 数字は AI が引用したくなる構造（曖昧さがない）。各セクション h3 直近に配置。
